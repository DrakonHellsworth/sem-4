Can abstract classes have constructors?
Yes, abstract classes can have constructors in Java even though you cannot create an object of an abstract class directly. This confuses a lot of people because they think if you can't instantiate it then what is the point of a constructor in it. The reason is that when a child class extends an abstract class and you create an object of the child class, the child class constructor automatically calls the parent abstract class constructor via `super()`, either explicitly or implicitly. So the constructor of the abstract class is not being called to create an abstract class object, it is being called to initialize the fields and setup the state that belongs to the abstract class portion of the child object. For example if an abstract class has some fields like `p`, `si`, `r` like in our Bank program, those fields need to be initialized somehow when a child like SBI or HDFC is created, and that initialization happens through the abstract class constructor when `super()` is called. So the constructor is basically there to serve the child classes, not to create objects of the abstract class itself. If you try to do `new AbstractClass()` directly Java will throw a compile time error, but the constructor still runs indirectly every time a child class object is created. So yes abstract classes can and often should have constructors for proper field initialization.
