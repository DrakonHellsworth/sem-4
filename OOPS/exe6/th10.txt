Difference between extends & implements.
Both `extends` and `implements` are used to achieve inheritance in Java but they are used in different situations. `extends` is used when a class wants to inherit from another class, or when an interface wants to inherit from another interface. With `extends` a class can only extend one class at a time because Java does not support multiple inheritance with classes. `implements` on the other hand is used when a class wants to implement an interface, and unlike `extends` a class can implement any number of interfaces at the same time separated by commas. The key difference in behavior is that when you `extend` a class you inherit both the method signatures and their actual implementations, so you get the full working methods from the parent. But when you `implement` an interface you only get the method signatures and you are forced to provide your own implementation for all of them, unless they are default methods. Another difference is that an interface cannot `implement` another interface, it can only `extend` another interface, while a class can never `extend` an interface, it can only `implement` it. So basically `extends` is class to class or interface to interface, and `implements` is class to interface, and the main practical difference is that `extends` gives you one parent with full implementation while `implements` lets you take from multiple interfaces but you have to write the implementations yourself.
