What are Default methods?
Default methods are methods defined inside an interface that have an actual body or implementation, introduced in Java 8. Before Java 8 interfaces could only have abstract methods meaning just the method signature with no body, and any class implementing the interface had to provide the implementation for all those methods. But the problem was that if you had an interface being implemented by hundreds of classes and you wanted to add a new method to that interface, every single class would break because now they all need to implement that new method. To solve this Java 8 introduced default methods where you can define a method directly inside the interface with the `default` keyword and give it a full implementation, and any class that implements the interface automatically inherits that implementation without being forced to override it. The class can still choose to override the default method if it wants its own version, but it is not mandatory. Now the diamond problem risk that was mentioned earlier comes in here, if two interfaces have a default method with the same name and a class implements both of them, Java forces that class to override the conflicting method and provide its own implementation to resolve the ambiguity, otherwise it is a compile time error. So basically default methods were added to allow interfaces to evolve and add new functionality over time without breaking all the existing classes that already implement them, it is more of a backward compatibility feature than anything else.
