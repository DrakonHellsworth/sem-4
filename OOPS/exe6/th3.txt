. What is the difference between this() and super() calls?
Both `this()` and `super()` are constructor calls but they serve different purposes. `this()` is used to call another constructor of the same class, basically when you have multiple constructors and you want one constructor to reuse the logic of another constructor in the same class, you use `this()`. On the other hand `super()` is used to call the constructor of the parent class from inside the child class constructor, which is what we used in our Bank program when SBI and HDFC called `super(p, si, 3.2)` to initialize the parent Bank's fields. One important rule for both is that they must always be the very first statement inside the constructor, and because of this you can never use both `this()` and `super()` together in the same constructor since only one can be the first statement. Also if you don't write `super()` explicitly, Java automatically inserts a default `super()` call at the start of every child class constructor, but if the parent has no default constructor then you have to call `super()` manually with the right arguments. So basically `this()` is same class constructor to constructor call, and `super()` is child to parent constructor call.
