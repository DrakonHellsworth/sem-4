. Why is Runtime Polymorphism also called "Late Binding"?

Runtime Polymorphism is also called Late Binding because the method call doesn't link to its definition at compile time — it happens late, at runtime. When a child object is assigned to a parent reference and an overridden method is called, the compiler only sees the reference type and can't decide which version to run. That decision is made at runtime, when the JVM sees the actual object and binds the call to the correct method — hence the name Late Binding or Dynamic Binding.
