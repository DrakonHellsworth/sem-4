. Why is Runtime Polymorphism also called "Late Binding"?

Runtime Polymorphism is also called "Late Binding" because the method call doesn't get linked to its actual definition at compile time, the binding happens late, that is at runtime. When we assign a child class object to a parent class reference and call an overridden method, the compiler only knows the reference type so it can't decide which version of the method to run at that point. It's only at runtime when the JVM sees the actual object that it binds the call to the correct overridden method. So basically since this linking is delayed till runtime, it is called Late Binding or Dynamic Binding.
