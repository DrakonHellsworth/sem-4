Can an interface extend another interface?
Yes an interface can extend another interface in Java and it uses the `extends` keyword for this, not `implements`. When an interface extends another interface it inherits all the abstract methods of the parent interface, and any class that implements the child interface is then responsible for providing implementations for all the methods, both the ones defined in the child interface and the ones inherited from the parent interface. This is different from a class extending another class because here there is no actual implementation being inherited, just method signatures, so there is no diamond problem risk with simple abstract methods. Also unlike classes, an interface can extend multiple interfaces at the same time, so you can write `interface C extends A, B` which is perfectly valid in Java. The diamond problem only becomes a concern when default methods are involved, if two parent interfaces have a default method with the same name and a third interface extends both of them, then any class implementing that third interface has to override the conflicting default method to resolve the ambiguity. This ability of interfaces to extend multiple interfaces is actually one of the reasons why Java uses interfaces to simulate multiple inheritance behavior that is not possible with classes. So basically interface extending interface is allowed, uses `extends`, can be multiple, and the implementing class at the end of the chain is responsible for implementing everything.
