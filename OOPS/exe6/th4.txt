What happens if you try to override a final method?
If you try to override a final method in Java, you will get a compile time error itself, Java will not even let the program run. The reason is that when you mark a method as `final`, you are basically telling the compiler that this method cannot be changed or overridden by any subclass, it is the final version of that method. So if a subclass tries to define a method with the same name and signature as a final method in the parent class, the compiler immediately throws an error saying something like "cannot override final method". This is actually done on purpose, `final` is used when you want to lock down a method's behavior and make sure no child class can mess with it. Same way if you mark an entire class as `final`, like the `String` class in Java, then that class cannot be extended at all. So basically `final` on a method means the method is closed for overriding, and any attempt to do so is a compile time error not even a runtime one.
