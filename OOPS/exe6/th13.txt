Can we define static methods in interfaces?
Yes, static methods are allowed in interfaces in Java and this was also introduced in Java 8 along with default methods. A static method in an interface has a full body and implementation just like a default method, but the key difference is that a static method belongs to the interface itself and not to any implementing class. This means you cannot call a static method of an interface using an object of the implementing class or through the implementing class name, you have to call it directly using the interface name like `InterfaceName.methodName()`. This is very different from default methods which are inherited by the implementing class and can be overridden, static methods in interfaces are not inherited at all by the implementing classes or even by sub interfaces that extend the parent interface. The reason they were added is basically to provide utility or helper methods that are related to the interface and make sense to keep inside the interface itself rather than creating a separate utility class for them. For example if you have a `Bank` interface you could have a static method inside it like a validator or a calculator that is general purpose and related to banking but does not need to be part of every implementing class. So to summarize, yes interfaces can have static methods since Java 8, they have a full body, they belong to the interface itself, they are called using the interface name directly, they are not inherited by implementing classes, and they are mainly used as utility or helper methods related to that interface.
