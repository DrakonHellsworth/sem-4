Explain Covariant Return Types in the context of method overriding.
Normally in method overriding the return type of the overriding method in the child class has to exactly match the return type of the method in the parent class. But covariant return type is an exception to this rule where the overriding method in the child class is allowed to return a subtype of the return type declared in the parent class. For example if the parent class method returns a Bank object, the child class can override it and return an SBI object instead, because SBI is a subclass of Bank so it is a more specific or narrower type. This was introduced in Java 5 and the idea behind it is pretty logical, if the parent says "I will return a Bank", the child saying "I will return an SBI" is still valid because SBI is a Bank, it is just more specific. This makes the code cleaner because without covariant return types you would have to return the parent type and then manually cast it to the child type every time you used it. One thing to remember is that this only works one way, the return type can get narrower meaning more specific going from parent to child, but it cannot get broader, so if the parent returns SBI the child cannot override it to return Bank because that would be a wider type and Java won't allow that. So basically covariant return type means the overriding method can return the same type or any subtype of the original return type.
