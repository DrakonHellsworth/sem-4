Is 'final' allowed for abstract classes?
No, `final` and `abstract` are completely opposite and contradictory keywords in Java so you cannot use them together on a class. The whole purpose of an abstract class is that it is meant to be extended by subclasses, it is incomplete on its own and relies on child classes to provide implementations for its abstract methods. On the other hand the whole purpose of `final` on a class is to prevent it from being extended by anyone. So if you try to write `final abstract class Something` Java will immediately give you a compile time error because you are basically saying "this class must be extended but also cannot be extended" at the same time which makes no sense. Same logic applies to methods as well, an `abstract` method has no body and exists only so that child classes can override and implement it, but a `final` method cannot be overridden at all, so `final abstract` on a method is also completely illegal and will give a compile time error. So the rule is simple, `final` and `abstract` can never be used together whether on a class or a method because their purposes are directly opposite to each other.
