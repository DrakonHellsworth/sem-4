What are Marker Interfaces?
A marker interface is an interface that has no methods and no fields inside it, it is completely empty. The only purpose of a marker interface is to mark or tag a class to signal to the JVM or the compiler that this class has some special property or should be treated in a special way. The most famous examples in Java are `Serializable` and `Cloneable`. When you implement `Serializable` on a class you are not forced to implement any method because there are none, you are simply telling the JVM that objects of this class are allowed to be serialized meaning converted to a byte stream and saved or transferred. If you try to serialize an object of a class that does not implement `Serializable`, the JVM will throw a `NotSerializableException` at runtime. So the interface itself does nothing functionally, it just acts as a flag or a label on the class. Same with `Cloneable`, implementing it tells the JVM that this class permits cloning, without it if you try to call `clone()` on an object you get a `CloneNotSupportedException`. Now with Java 5 annotations came along and many people argue that annotations are a better and more flexible way to tag classes compared to marker interfaces, and that is true to some extent, but marker interfaces are still used in Java and are still an important concept to know. So basically a marker interface is just an empty interface used to tag a class and convey metadata to the JVM about how objects of that class should be handled.
